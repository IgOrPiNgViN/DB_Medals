# ООН ПКР — База данных «Отраслевые общественные награды промышленного комплекса России»

Клиент-серверное приложение для управления базой данных наград, лауреатов и наградного комитета.

## Архитектура

- **Сервер**: FastAPI + SQLAlchemy + PostgreSQL
- **Клиент**: PyQt5 (десктопное приложение)
- **Связь**: REST API через HTTP

## Требования

- Python 3.10+
- PostgreSQL 15+

## Установка и запуск

### 1. Установка PostgreSQL

Скачайте и установите PostgreSQL: https://www.postgresql.org/download/windows/

Создайте базу данных:
```sql
CREATE DATABASE awards_db;
```

### 2. Запуск сервера (API, без окна)

Это **только бэкенд** — в браузере откроется только документация, если вы её откроете сами. Окно программы не появится.

```bash
cd server
pip install -r requirements.txt

# Настройте подключение к БД (по умолчанию: postgresql://postgres:1234@localhost:5432/awards_db)
# Можно задать переменную окружения:
# set DATABASE_URL=postgresql://user:password@host:port/awards_db

python main.py
```

Сервер: http://localhost:8000 — Swagger: http://localhost:8000/docs

### 3. Запуск клиента (окно приложения PyQt5)

**Откройте второй терминал** (сервер из шага 2 должен продолжать работать).

```bash
cd client
pip install -r requirements.txt

# Если сервер на другом адресе:
# set SERVER_URL=http://192.168.1.100:8000

python main.py
```

Именно эта команда открывает **десктопное приложение** с боковым меню.

**Ошибка «Could not find the Qt platform plugin windows»**  
В `client/main.py` при старте выставляются пути к плагинам PyQt5. Если ошибка остаётся, в PowerShell укажите каталог вручную (путь подставьте свой, где установлен Python):

```powershell
$py = python -c "import PyQt5, os; print(os.path.dirname(PyQt5.__file__))"
$env:QT_PLUGIN_PATH = "$py\Qt5\plugins"
$env:QT_QPA_PLATFORM_PLUGIN_PATH = "$py\Qt5\plugins\platforms"
python main.py
```

Либо переустановите PyQt5: `pip uninstall PyQt5 PyQt5-Qt5 PyQt5-sip -y` и `pip install PyQt5`.

## Разделы приложения

### Награды
- Карточки наград (создание, удаление, фильтрация по типу)
- Детали: характеристика, учреждение, разработка, согласование, производство
- Жизненный цикл наград (сводный отчёт по 4 типам наград)
- Склад (физический учёт с оповещениями при остатке < 10 шт.)

### Лауреаты
- Карточки лауреатов с категориями (сотрудники, ветераны, университеты, НИИ, орг.)
- Привязка лауреата к награде
- ЖЦ лауреата (6 этапов: выдвижение → голосование → решение → оформление → вручение → опубликование)
- Отчёты: награды-лауреаты, незавершённый ЖЦ, **этапы ЖЦ** (сколько на каждом этапе), статистика

### Наградной комитет
- Список членов НК (действующие/не действующие)
- Карточки с правами подписи и полномочиями
- Назначение подписантом удостоверений / уполномоченным по награде

### Голосование
- Создание бюллетеней (разделы, вопросы, рассылка)
- Мониторинг ответов (цветовая индикация: зелёный/красный/серый)
- Подсчёт голосов (порог принятия решения: 65%)
- Формирование протоколов
- Выписки из протоколов
- Представления на награждение ППЗ

### Сервис
- Полный бэкап PostgreSQL (pg_dump)
- Восстановление из дампа
- Экспорт отдельных таблиц в CSV

## Миграция данных из Access

### Почему в приложении «пусто»

Новая БД **PostgreSQL** изначально **без строк**: склад, лауреаты и т.д. заполняются после **импорта** или **ввода вручную**. Это не баг интерфейса.

Присланные файлы `.accdb` — **часто только оболочка**: таблицы **связаны** с другим файлом на диске организации (`Z:\...`). Пока этот **back-end** не лежит у вас на диске и связи не переподключены, **прочитать данные** из Access с вашего ПК нельзя — выгрузка даст ошибку пути.

### Шаг 1 — получить данные у организации

Запросите один из вариантов:

- файл **back-end** `.accdb` (где лежат таблицы), с инструкцией переподключить связи на локальный путь;  
- или **полную** базу без внешних связей;  
- или **выгрузку Excel/CSV** по таблицам с их стороны.

### Шаг 2 — выгрузка из Access в CSV (когда файл с данными открывается)

```powershell
$env:ACCDB_PATH = "C:\путь\к\файлу_с_таблицами.accdb"
$env:OUT_DIR = "migration\csv_export"
python migration\dump_access_to_csv.py
```

CSV появятся в `migration\csv_export\`.

### Шаг 3 — импорт в PostgreSQL (чтобы данные появились в приложении)

Скрипт **`migration/import_from_csv.py`** очищает пользовательские таблицы в PostgreSQL и загружает из CSV основное: награды (`НаградыМега.csv`), лауреаты (`Лауреаты.csv`), связи и этапы ЖЦ (`ЛАУР_ЖЦ.csv`), члены НК (`Список НК.csv`), склад (`УЧ_комплекты_наград.csv`).

Убедитесь, что строка подключения совпадает с `server/config.py` (или задайте `DATABASE_URL`). Затем из **корня репозитория**:

```powershell
python migration\import_from_csv.py
```

После этого перезапустите **сервер** FastAPI и **клиент** PyQt — разделы «Награды», «Склад», «Лауреаты», «НК» должны показывать импортированные строки.

Импорт также заполняет таблицу **`access_mirror_rows`** — полная копия **каждого** CSV (все столбцы и строки, как в бэкенде Access). В клиенте откройте **Сервис → Таблицы Access (как в бэкенде)** — выберите имя таблицы и нажмите **Показать**, чтобы увидеть то же содержимое, что в `.accdb` после выгрузки.

Импорт **не** переносит бюллетени, голоса и вложения из Access целиком — только перечисленные сущности. Остальное можно доработать отдельно.

Колонки **НаградыМега** дополнительно пишутся в таблицу `award_characteristics` и отображаются в клиенте на вкладке награды **«Характеристика»** (сгруппировано похоже на блоки формы Access). Если база уже была создана раньше, при ошибке длины имени поля выполните в PostgreSQL:

```sql
ALTER TABLE award_characteristics ALTER COLUMN field_name TYPE TEXT;
```
